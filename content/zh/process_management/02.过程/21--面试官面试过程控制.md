为了避免面试官的主观意向与知识体系限制，需要为每个岗位制定不同的面试问题与大纲。为了达到更好的效果可以将面试内容分成几个部分：公共部分，通用能力部分，岗位能力部分，最终评级。

#### 公共部分：
主要记录一些人员背景，可以为了解面试者提供最简要描述。
```
1. 背景：
	工作经验：
	学校： 专业：
	并发：   机器数量：
```

#### 通用能力部分：
```
2. 职业能力：
    1. 沟通：
    2. 逻辑：
    3. 态度：
    4. 学习：
```

#### 岗位能力部分：
因每个岗位都不尽相同，一方面需要考量面试者的专业技能，另一方面要考量专业的思维体系是否完善。例如运维人员的岗位能力：
```
4. 发展方向：
5. 管理：
6. 运维知识体系：
    1. 运维体系：
        1. 事故管理（Incident management）：
        2. 问题管理（Problem management）：
        3. 配置管理（Configuration Management）：
        4. 变更管理（Change Management）：
        5. 发布管理（Release Management）：
    2. 业务指标体系：
    3. 技术指标体系：
    4. 容量规划：
    5. Devops：
7. 系统性思维：
8. 运维知识：
    1. 安全扫描：
    2. 操作系统：
    3. 配置管理：
    4. 巡检：
    5. 监控：
        1. 微服务监控：
    6. Web服务器：
    7. 持续集成：
    8. 中间件：
    9. 虚拟化：
    10. 数据库：
        1. Sql审计：
        2. 数据库升级：
    11. 日志集中化分析：
    12. 网络：
    13. 存储：
    14. 堡垒机：
```
